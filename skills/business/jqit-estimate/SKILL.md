# JQIT 見積書自動生成

案件情報を入力するだけで、JQIT標準フォーマットの見積書xlsxを生成する。

## 入力→出力

```
入力: 案件名、顧客名、スコープ概要（or requirement.md）
  ↓ 工数自動算出 + 価格設定
出力: 御見積書.xlsx（印刷可能、A4縦）
```

## ワークフロー

### Step 1: 案件情報の取得

以下の情報を収集する（対話 or 既存ドキュメントから抽出）:

**必須:**
- 顧客名（会社名）
- 件名（案件名）
- スコープ概要（何を作るか）

**任意（デフォルトあり）:**
- 顧客担当者名
- 見積日（デフォルト: 今日）
- 有効期限（デフォルト: 30日後）
- 支払条件（デフォルト: 着手金50%+残金50%）
- 納期目安

**ai-solution-demoプロジェクトがある場合:**
- `docs/requirement.md` から要件を自動抽出
- `docs/memo.md` からスコープを把握
- これらを読み込んで工数を自動算出

### Step 2: 工数算出と明細作成

[references/pricing-guide.md](references/pricing-guide.md) の単価・工数目安を参照し、明細を自動生成。

**算出ルール:**
1. スコープから必要な作業項目を特定
2. 各項目の工数を pricing-guide の目安から算出
3. 案件規模に応じた項目テンプレートを選択（小規模/中規模）
4. AI関連機能があればオプション項目を追加
5. 合計が100万〜300万円のレンジに収まるよう調整

**明細を生成したら、ユーザーに提示して確認を取る。** 金額の調整はここで行う。

### Step 3: xlsx生成

document-skills:xlsx スキルを使用して見積書xlsxを生成する。

[references/estimate-format.md](references/estimate-format.md) のフォーマット仕様に従い、以下の構成で生成:

**シート1: 御見積書**
```
【A1-F1】     タイトル「御見積書」
【右上】       見積番号、見積日、有効期限
【件名行】     案件名
【左】         宛先（顧客名 御中）
【合計金額】   税込合計（大きく表示）
【明細テーブル】 No./項目名/数量/単位/単価/金額
【小計行】     小計/消費税/合計
【備考欄】     支払条件、納期、前提条件（「スコープは別紙のとおり」を明記）
【発行者】     JQIT社名・住所・代表者
```

**シート2: 別紙_スコープ定義（必須）**

見積書には必ず「別紙_スコープ定義」シートを付ける。これにより見積範囲を明確化し、スコープクリープを防ぐ。

```
【タイトル】   別紙：スコープ定義書
【サブタイトル】 案件名 — 御見積書 EST-XXXXXXXX-NNN 対応

■ 本見積に含まれる機能（スコープ内）
  No. / 機能カテゴリ / 機能名 / 概要 / 対応フェーズ

■ 本見積に含まれない機能（スコープ外）
  No. / 機能カテゴリ / 機能名 / 備考（除外理由）/ 対応フェーズ

■ 判明している要件・前提条件
  No. / カテゴリ / 要件・前提 / 備考 / ステータス（確定 or 仮説（要検証））
```

**スコープ定義の情報源:**
- `docs/01-requirements/requirements.md` の機能要件（F-xxx）→ スコープ内
- `docs/01-requirements/requirements.md` の Out of Scope → スコープ外
- `docs/02-design/design.md` の技術スタック → 前提条件
- `docs/memo.md` の案件背景 → 前提条件

**備考欄には以下を記載:**
```
【スコープ】
別紙「スコープ定義書」のとおり。
```

**スタイル:**
- ヘッダー背景: #1B3A5C（濃紺）、文字白
- フォント: メイリオ or Yu Gothic
- 印刷範囲・A4縦設定
- 金額はカンマ区切り、¥マーク付き

### Step 4: 出力と案内

1. xlsxファイルを保存
2. ファイルパスをユーザーに案内
3. 次のステップを提案:
   - 「内容を確認して、修正があればお知らせください」
   - 「PDF化する場合: `soffice --headless --convert-to pdf 見積書.xlsx`」
   - 「契約書の準備が必要な場合は `/legal-review` で基本契約書のチェックもできます」

## ai-solution-demo との連携

ai-solution-demo（Phase 9: 提案書）の後続フローとして使用する場合:

```
/ai-solution-demo phase 9  → 提案書PPTX完成
↓
/jqit-estimate             → 見積書xlsx生成
↓
/legal-review              → 基本契約書・NDAのチェック
↓
契約締結 → 着手金請求 → 開発開始
```

プロジェクトディレクトリに `docs/requirement.md` があれば自動的に読み込み、工数算出の精度が上がる。
