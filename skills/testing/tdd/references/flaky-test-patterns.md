# フレーキーテスト対策

E2Eテストの不安定性（flakiness）への対処パターンと Playwright Healer Agent 連携ガイド。

---

## よくあるフレーキー原因と対策

| 原因 | 症状 | 対策 |
|------|------|------|
| **タイミング依存** | 要素が見つからない、クリックできない | `waitForSelector` / `expect().toBeVisible()` で明示的に待機 |
| **アニメーション干渉** | クリックが別要素に当たる | `animations: "disabled"` or `force: true` |
| **テスト間の状態汚染** | 前テストのデータが残る | `beforeEach` でクリーンアップ、独立したテストデータ |
| **ネットワーク遅延** | API応答のタイムアウト | `page.waitForResponse()` で明示的に待機 |
| **ポップアップ・モーダル** | 予期しないオーバーレイ | テスト開始時に dismissed 確認 |
| **セレクタ脆弱性** | DOM構造変更で壊れる | `getByRole` / `getByTestId` を優先 |

---

## Healer Agent 連携

Playwright Healer Agent はテスト失敗を自動診断・修復する。

### 自動修復の対象

- **セレクタ変更**: DOM構造変更に追随してセレクタを更新
- **タイミング問題**: 適切な待機処理を自動挿入
- **アサーション値変更**: 実際の値に基づいてアサーションを更新

### 使い方

1. テスト失敗 → `test_run` で再実行し失敗を再現
2. `test_debug` で失敗箇所を診断
3. Healer が自動修正を提案 → 適用 → 再実行

### エスカレーション条件

- Healer が `test.fixme()` マーク → 人間に報告（自動修復不能）
- 同じテストが3回連続でフレーキー → 根本原因調査が必要

---

## クォランティン戦略

繰り返しフレーキーなテストは隔離して対処する。

1. **タグ付け**: `test.describe.configure({ retries: 2 })` で自動リトライ
2. **隔離**: `test.fixme()` で一時スキップ（CI を止めない）
3. **追跡**: 隔離したテストを Issue/チケットで管理
4. **期限**: 隔離から2週間以内に修正 or 削除
