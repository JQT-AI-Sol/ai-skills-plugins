# シナリオ設計ガイド

E2Eテストのシナリオ設計に使用する。Playwright Planner Agent と連携して効率的にシナリオを作成する。

---

## シナリオ分類（6カテゴリ）

| # | カテゴリ | 目的 | 例 |
|---|---------|------|-----|
| 1 | **ハッピーパス** | 正常フローが意図通り動くか | フォーム入力→送信→完了表示 |
| 2 | **異常系・エラー** | エラー状態の表示が適切か | バリデーションエラー、API失敗、404 |
| 3 | **画面状態バリエーション** | 各データ状態でUIが崩れないか | 空状態、大量データ、ローディング中 |
| 4 | **データストレス** | 極端なデータで表示が壊れないか | 長文、特殊文字、0件、100件超 |
| 5 | **画面遷移・ナビゲーション** | ページ間の遷移が自然か | ブラウザバック、ディープリンク、タブ切替 |
| 6 | **権限・ロール別** | ロールごとの表示差異が正しいか | 管理者/一般ユーザー/未認証 |

---

## 優先度ルール

1. **変更した画面のハッピーパス** — 必ずテスト
2. **変更した画面の異常系** — UIに影響する変更なら追加
3. **変更の影響を受ける隣接画面** — 共通コンポーネント変更時
4. **画面状態バリエーション** — 新規画面・大きなレイアウト変更時
5. **データストレス** — フォーム・テーブル・リスト表示の変更時
6. **権限・ロール別** — 認可ロジック変更時

---

## Playwright Planner Agent 連携

Planner Agent を使ってテストシナリオを自動探索・生成する。

### 使い方

1. **UI探索**: `planner_setup_page` で対象ページを開く
2. **アクセシビリティツリー探索**: Planner がページ構造を分析し、操作可能な要素を特定
3. **シナリオ生成**: `planner_save_plan` で探索結果をテスト計画として保存
4. **回帰シナリオ**: Planner が既存テストと変更差分から回帰テストシナリオを提案

### Planner が生成するもの

- 操作手順のステップリスト
- 各ステップの期待結果
- アクセシビリティ観点のテスト項目（ARIA属性、キーボードナビゲーション）

### 注意点

- Planner の出力はあくまで提案。人間が最終判断する
- 複雑なビジネスロジックのシナリオは手動で補完する
- 生成されたシナリオは必ず POM 経由のコードに変換する
