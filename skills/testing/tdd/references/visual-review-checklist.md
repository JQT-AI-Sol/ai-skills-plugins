# 目視確認チェックリスト（15項目）

E2Eテスト後のUI目視レビューで使用する。動画を見ながら、**変更に関連する項目**を重点的に確認する。

---

## 1. レイアウト・構造（必ず確認）

- [ ] 要素の配置・整列が意図通りか（左寄せ、中央、グリッド）
- [ ] 余白（padding/margin）が均一で、詰まりすぎ・開きすぎがないか
- [ ] コンテンツが画面幅に収まっているか（横スクロールが出ていないか）

## 2. テキスト・タイポグラフィ

- [ ] フォントサイズの階層が適切か（見出し > 本文 > 補足）
- [ ] テキストが途中で切れたり、はみ出したりしていないか
- [ ] 長文テキストの折り返し・省略（...）が自然か

## 3. 色・コントラスト

- [ ] ボタンやリンクの色がクリック可能に見えるか
- [ ] エラー/警告/成功の色分けが直感的か（赤=エラー、緑=成功）
- [ ] テキストと背景のコントラストが十分か

## 4. インタラクション・フィードバック

- [ ] クリック後の応答がある（ローディング表示、ボタン無効化）
- [ ] 画面遷移のアニメーションが自然か（ちらつき、白画面がないか）
- [ ] フォーム送信後の成功/失敗のフィードバックが明確か

## 5. 状態表示

- [ ] データ0件時の空状態メッセージが分かりやすいか
- [ ] ローディング中にスピナーやスケルトンが表示されるか
- [ ] エラー発生時にユーザーが次に何をすべきか分かるか（リトライボタン等）
