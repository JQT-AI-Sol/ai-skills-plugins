# テストデータパターン

E2Eテスト・API統合テストで使用するストレスデータと画面状態パターン。

---

## ストレスデータ（8パターン）

| データパターン | 値の例 | 壊れやすいUI |
|--------------|-------|------------|
| **空文字** | `""` | プレースホルダー消失、高さ0 |
| **長文（100文字超）** | `"あ"×120` | テーブルセル溢れ、カード破裂 |
| **特殊文字** | `<script>alert(1)</script>` `"引用"` `&amp;` | XSS、エスケープ漏れ |
| **数値境界** | `0`, `-1`, `999999999` | 桁あふれ、マイナス表示 |
| **日本語+絵文字** | `テスト🎉確認✅` | フォント崩れ、切り詰め |
| **0件データ** | 空配列 | 空状態メッセージ未表示 |
| **大量データ（50件超）** | ダミー50-100件 | ページネーション、スクロール |
| **画像なし** | `null` / 壊れたURL | alt表示、プレースホルダー |

---

## 画面状態マトリクス（6状態）

| 状態 | 再現方法 | 確認ポイント |
|------|---------|------------|
| **初期表示（空状態）** | データなしでページ遷移 | 空状態メッセージ、CTA導線 |
| **ローディング中** | `page.route()` でAPI遅延注入 | スピナー/スケルトン表示 |
| **正常データ表示** | 標準テストデータ | レイアウト、余白、整列 |
| **大量データ表示** | 50件超のテストデータ | スクロール、ページネーション |
| **エラー状態** | `page.route()` で500返却 | エラーUI、リトライ導線 |
| **部分エラー** | 一部APIのみ失敗 | 正常部分の表示、エラー部分の分離 |

---

## APIモックパターン

```typescript
// 遅延レスポンス（ローディングUI確認用）
await page.route("**/api/data", async (route) => {
  await new Promise((r) => setTimeout(r, 3000));
  await route.fulfill({ json: { items: [] } });
});

// エラーレスポンス
await page.route("**/api/data", (route) =>
  route.fulfill({ status: 500, json: { error: "Internal Server Error" } })
);

// 空データ
await page.route("**/api/data", (route) =>
  route.fulfill({ json: { items: [], total: 0 } })
);
```
