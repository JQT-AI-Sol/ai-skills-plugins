# TDD UI Review（Gate 7: UI/UXレビュー＆改善ループ）

> **ロール**: Generator（スクショ撮影・修正実装）→ Evaluator（3段階優先度スコアリング・Gate 7 判定）

E2Eテスト対象画面のUI/UX品質を、Playwright MCPスクリーンショット + レビュースキルで段階的に改善する。

---

## スキップ条件

- 単体テスト・API統合テストのみの変更
- ユーザーが「UI確認不要」と明示的に指示

---

## Step 1: スクリーンショット撮影（Playwright MCP）

変更対象の全画面 + 主要な操作状態のスクリーンショットを撮影する。

**撮影対象:**
- 各ページの初期表示状態
- データあり / データなし（空状態）の両方
- モーダル・ドロワー・ツールチップの展開状態
- レスポンシブ: デスクトップ（1280px）+ モバイル（375px）

**手順:**
1. `mcp__playwright__browser_navigate` で対象ページを開く
2. `mcp__playwright__browser_take_screenshot` で各状態をキャプチャ
3. スクリーンショットを確認し、Step 2 で評価

---

## Step 2: UI/UX評価（3段階の優先度）

### Priority 1: オリジナリティ・AI臭さ・デザイン統一性 [CRITICAL]

> **「AIが作りました」と一目で分かるUIは不合格。ページ間でコンポーネントがバラバラなUIも不合格。**

`/review-vercel-frontend`（ui-ux-pro-max統合）でチェックする。

#### 1a. オリジナリティ＆AI臭さ

| チェック項目 | NG例 | 改善方向 |
|------------|------|---------|
| テンプレ感 | shadcn/uiのデフォルトそのまま、どこかで見たレイアウト | カラー・スペーシング・タイポグラフィをプロジェクト固有にカスタマイズ |
| 均一すぎるデザイン | 全要素が同じサイズ・同じ余白・同じ角丸 | 視覚的階層（大きさ・色・余白の差）でメリハリをつける |
| ストック画像感 | 無意味なイラスト・アイコンの多用 | 必要最小限のアイコン、データドリブンなビジュアル |
| 過剰な装飾 | 不要なグラデーション・影・アニメーション | 情報伝達に必要な装飾だけ残す |
| ジェネリックなコピー | 「Welcome to our platform」的な汎用テキスト | プロダクト固有の言葉遣い |

#### 1b. デザインシステム統一性

> 全ページのスクリーンショットを並べて比較し、システム全体の一貫性を検証する。

| チェック項目 | 確認ポイント |
|------------|------------|
| ボタンの統一 | Primary/Secondary/Destructiveの色・サイズ・角丸・パディングが全ページで同じか |
| 入力フォームの統一 | Input/Select/Textarea/Checkbox のスタイル・バリデーション表示が揃っているか |
| カラーパレット | ブランドカラー・アクセントカラー・背景色がページ間で統一されているか |
| タイポグラフィ | 見出し（H1-H4）・本文・ラベル・キャプションのフォントサイズ・ウェイトが一貫しているか |
| スペーシング | セクション間・カード間・要素間の余白ルールが全ページで統一されているか |
| アイコンスタイル | アイコンのサイズ・線幅・スタイル（Outline/Filled）が混在していないか |
| ナビゲーションパターン | ページ間の遷移パターン（戻るボタン、パンくず、タブ）が統一されているか |
| テーブル・リスト表示 | データ一覧の表示パターン（テーブル/カード/リスト）がコンテキストに合って統一されているか |
| モーダル・トースト | 確認ダイアログ・通知トーストの見た目と挙動が全箇所で同じか |

### Priority 2: レイアウト＆ビジュアル品質

| チェック項目 | 確認ポイント |
|------------|------------|
| 文字の見切れ | 長いテキストが省略なく収まっているか、tooltipで補完されているか |
| 余白バランス | 要素間のスペーシングが一定か、窮屈/スカスカな箇所がないか |
| レスポンシブ崩れ | モバイル表示で重なり・はみ出し・横スクロールがないか |
| コントラスト比 | テキストが背景に対して読みやすいか（WCAG AA 4.5:1以上） |
| フォント一貫性 | フォントファミリー・サイズ・ウェイトがデザインシステムに沿っているか |
| ダークモード対応 | ダークモード時に色が崩れていないか（対応している場合） |

### Priority 3: UX導線＆ユーザー体験

| チェック項目 | 確認ポイント |
|------------|------------|
| 操作ステップ数 | 主要タスクが最小ステップで完了できるか（3クリック以内が目安） |
| 迷わない導線 | 次に何をすべきかが画面を見て即座に分かるか |
| 不要な確認 | 不要なモーダル確認・余計な入力項目がないか |
| エラー時のガイド | エラー時に「何が悪くて、どうすれば直るか」が表示されるか |
| ローディング状態 | 非同期処理中にスケルトン/スピナーが表示されるか |
| 空状態 | データがない時に「何をすべきか」のガイドがあるか |
| フォーカス順序 | Tab キーの移動順序が自然か |

---

## Step 3: 改善イテレーション

```
スクショ撮影 → 評価 → 指摘リスト作成 → 修正 → 再スクショ → 再評価
                                                    ↑
                                        Gate 7 PASS まで繰り返す
```

**手順:**
1. Step 2 の評価結果から指摘リストを作成
2. Priority 1（CRITICAL）の指摘を先に全て修正
3. 修正後、Playwright MCP で再度スクリーンショットを撮影
4. Priority 1 がクリアになったら Priority 2 → 3 の順に修正
5. 全指摘の修正後、Gate 7 評価に進む

**イテレーション上限**: 最大3ラウンド。3ラウンドでCRITICALが解消しない場合はユーザーに相談。

## Gate 7 評価（Evaluator）

| 評価項目 | 配点 |
|---------|------|
| `[CRITICAL]` オリジナリティ（AI臭さゼロ） | 3 |
| `[CRITICAL]` テンプレ感の排除 | 3 |
| `[CRITICAL]` デザインシステム統一性（ボタン・フォーム・カラー・タイポグラフィ） | 3 |
| レイアウト崩れなし | 3 |
| レスポンシブ対応 | 3 |
| 文字の見切れ・バランス | 3 |
| UX導線の明快さ | 3 |
| 操作ステップの最小化 | 3 |
| エラー・空状態・ローディング | 3 |

**満点27点 / 合格ライン22点（80%）** — Critical項目0点で即FAIL
