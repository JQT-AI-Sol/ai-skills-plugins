## 1. 過剰な丁寧表現

| AI表現 | 人間らしい表現 |
|--------|--------------|
| 〜することが可能です | 〜できる |
| 〜を実現することができます | 〜できる |
| 〜を行うことが重要です | 〜が重要だ / 〜する |
| 〜について説明いたします | 〜を説明する |
| 〜という点が挙げられます | 〜がある |
| 〜と言えるでしょう | 〜だ |
| 〜であると考えられます | 〜だ / 〜だろう |
| 〜に関して述べますと | 〜は |

## 2. 冗長な接続・前置き

| AI表現 | 人間らしい表現 |
|--------|--------------|
| まず最初に | まず |
| それでは次に | 次に |
| ここで重要なのは | （直接述べる） |
| 以下に詳しく説明します | （直接説明に入る） |
| この点について補足すると | 補足すると |
| 上記を踏まえた上で | これを踏まえ |
| 結論から申し上げますと | 結論として |

## 3. 無意味な強調・修飾

| AI表現 | 人間らしい表現 |
|--------|--------------|
| 非常に重要な | 重要な（本当に強調が必要な場合のみ残す） |
| 大幅に向上させる | 向上させる |
| 包括的なアプローチ | アプローチ（具体性がなければ削除） |
| シームレスに統合 | 統合 |
| 効果的に活用する | 使う |
| 最適な〜を実現 | 〜を実現 |
| 革新的な | （具体的に何が新しいか書く） |
| 堅牢な | （具体的な強度を書く） |

## 5. AI特有のパターン

### 箇条書き病
- 全てを箇条書きにする → 文章で流せるものは文章にする
- 1項目しかない箇条書き → 文章に変換

### エコー（繰り返し）
- 導入で述べた内容を結論でほぼ同じ文で繰り返す
- セクション冒頭で前セクションの要約を繰り返す

### 過剰な構造化
- 2-3文で済む内容にH2/H3の見出しを付ける
- 「概要」「詳細」「まとめ」の3段構成で同じことを3回言う

### バズワードの乱用
- 「DX」「イノベーション」「パラダイムシフト」等を具体性なく使用
- 形容詞の羅列（「革新的で包括的な最先端のソリューション」）

## 6. 構造的な冗長パターン

### 同義セクションの重複
同じ内容を別の見出しで繰り返すパターン:
- 「背景」と「課題」で同じことを書く
- 「目的」と「ゴール」が実質同一
- 「効果」と「メリット」が同じ内容

### 不要なセクション
- 「はじめに」が本文の要約になっている（本文を読めばわかる）
- 「まとめ」が本文の再掲（新しい情報がない）
- 「参考」が空、または本文中のリンクの再掲
